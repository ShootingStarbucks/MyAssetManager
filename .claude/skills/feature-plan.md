# feature-plan — SKILLS.md 기반 기능 구현 계획

새 기능을 구현하기 전에 SKILLS.md 명세와 SKILLS.md §8 로드맵을 기준으로
구체적인 구현 계획을 수립합니다.

## 입력

사용자가 구현하려는 기능명 또는 Phase 번호를 제공합니다.
예: `/feature-plan 리스크 등급` 또는 `/feature-plan phase2`

## 계획 수립 절차

1. **명세 확인**: SKILLS.md에서 해당 기능의 정확한 스펙을 읽기
2. **현재 상태 파악**: 관련 파일을 읽어 현재 구현 수준 확인
3. **의존성 체크**: 이 기능이 완성되려면 먼저 필요한 것이 있는지 확인
4. **작업 분해**: 아래 형식으로 구체적 태스크 목록 작성
5. **브랜치 전략 제안**: CLAUDE.md Git Flow 규칙에 따라 브랜치명 제안

## 출력 형식

```
## 구현 계획: {기능명}

### SKILLS.md 명세 요약
- 규칙 1: ...
- 규칙 2: ...

### 현재 상태
- 관련 파일: src/lib/...
- 현재 구현: ...
- 부족한 부분: ...

### 선행 작업 (없으면 생략)
- [ ] ...이 먼저 필요합니다

### 구현 태스크
- [ ] 1. DB 스키마 변경 (있을 경우)
- [ ] 2. 계산 로직 추가/수정
- [ ] 3. API 라우트 추가/수정
- [ ] 4. UI 컴포넌트 추가/수정
- [ ] 5. 타입 정의 추가

### Git 브랜치
git checkout -b feature/{브랜치명} develop

### 완료 기준 (SKILLS.md 기준)
- [ ] SKILLS.md §X의 Y 조건 충족
- [ ] SKILLS_GAP_ANALYSIS.md 해당 항목 ✅ 로 변경
```

계획 확인 후 구현을 시작할지 사용자에게 확인하세요.
