## 1. 자산 분류 기준

### 1-1. 자산군 정의

| 자산군 | 설명 | 예시 |
|---|---|---|
| 주식 (STOCK) | 국내외 상장 주식 | 삼성전자, 애플, 테슬라 |
| 코인 (CRYPTO) | 암호화폐 | 비트코인(BTC), 이더리움(ETH) |
| 현금 (CASH) | 현금, 예금, CMA 등 유동성 자산 | 국민은행 정기예금, CMA |

### 1-2. 자산군별 입력 필드

**주식 (STOCK)**
```
- symbol: 티커 (예: 005930, AAPL)
- name: 종목명 (예: 삼성전자, Apple)
- exchange: 거래소 (KOSPI | KOSDAQ | NASDAQ | NYSE | 기타)
- quantity: 보유 수량 (양의 정수)
- purchasePrice: 평균 매수가 (KRW 또는 USD)
- currency: 통화 (KRW | USD)
- purchaseDate: 최초 매수일 (YYYY-MM-DD)
- currentPrice: 현재가 (사용자 직접 입력 또는 API 연동)
- memo: 메모 (선택)
```

**코인 (CRYPTO)**
```
- symbol: 심볼 (예: BTC, ETH)
- name: 코인명 (예: 비트코인, 이더리움)
- exchange: 거래소 (Upbit | Bithumb | Binance | 기타)
- quantity: 보유 수량 (소수점 8자리까지 허용)
- purchasePrice: 평균 매수가 (KRW 또는 USD)
- currency: 통화 (KRW | USD)
- purchaseDate: 최초 매수일 (YYYY-MM-DD)
- currentPrice: 현재가 (사용자 직접 입력 또는 API 연동)
- memo: 메모 (선택)
```

**현금 / 예금 (CASH)**
```
- institution: 금융기관명 (예: 국민은행)
- accountType: 계좌 유형 (입출금 | 정기예금 | CMA | 적금 | 기타)
- amount: 현재 잔액 (KRW)
- interestRate: 연이율 % (선택, 정기예금/적금의 경우)
- maturityDate: 만기일 (선택, YYYY-MM-DD)
- memo: 메모 (선택)
```

## 2. 투자 지표 계산 규칙

### 2-1. 기본 수익률 지표

**평가금액 (Evaluation Amount)**
```
주식/코인: 평가금액 = 현재가 × 보유수량
현금: 평가금액 = 잔액 (이자 미포함)
```

**평가손익 (Profit & Loss)**
```
평가손익 = 평가금액 - (매수가 × 보유수량)
```

**수익률 (Return Rate)**
```
수익률(%) = (평가손익 / (매수가 × 보유수량)) × 100
소수점 2자리까지 표시
양수: 수익 (초록색), 음수: 손실 (빨간색)
```

**총 자산 (Total Assets)**
```
총 자산 = Σ(모든 자산군의 평가금액)
단, 통화가 다를 경우 USD → KRW 환율 적용 (사용자 입력 또는 API)
```

**총 수익률 (Total Return Rate)**
```
총 수익률(%) = (Σ평가손익 / Σ매수금액) × 100
```

### 2-2. 자산 비중 계산

**자산군별 비중**
```
자산군 비중(%) = (해당 자산군 평가금액 합계 / 총 자산) × 100
```

**종목별 비중**
```
종목 비중(%) = (해당 종목 평가금액 / 총 자산) × 100
```

### 2-3. 리스크 지표

**자산 편중도 (Concentration)**
```
단일 종목 비중이 30% 초과 시 → 편중 경고 (WARNING)
단일 종목 비중이 50% 초과 시 → 고위험 경고 (DANGER)
단일 자산군(주식 또는 코인)이 80% 초과 시 → 자산군 편중 경고
```

**변동성 기반 리스크 등급**
```
코인 비중이 50% 이상 → 공격적 (AGGRESSIVE)
코인 비중이 30~49% 또는 주식 비중이 70% 이상 → 중립 (MODERATE)
현금 비중이 50% 이상 또는 코인 비중이 30% 미만 → 보수적 (CONSERVATIVE)
```

**샤프 지수 (Sharpe Ratio)**
```
샤프 지수 = (포트폴리오 수익률 - 무위험 수익률) / 포트폴리오 변동성
무위험 수익률 기준: 한국 국채 3년물 기준 (기본값 3.5%, 사용자 변경 가능)
변동성: 월별 수익률의 표준편차 × √12 (연환산)
데이터가 3개월 미만일 경우 샤프 지수 계산 불가 → "데이터 부족" 표시

샤프 지수 해석:
  1.0 이상 → 우수 (양호한 위험 대비 수익)
  0.5 ~ 1.0 → 보통
  0.5 미만 → 개선 필요
  음수 → 무위험 자산보다 낮은 성과
```

### 2-4. 리밸런싱 제안 계산

**목표 비중 대비 현재 비중 이탈 감지**
```
이탈 기준: 목표 비중 대비 ±10%p 초과 시 리밸런싱 제안
예시: 목표 주식 비중 60%, 현재 75% → +15%p 이탈 → 리밸런싱 필요

리밸런싱 제안 메시지 형식:
  "[종목/자산군]을 [현재비중]%에서 [목표비중]%로 조정하세요.
   약 [금액]원어치를 [매도/매수]하면 목표 비중에 도달합니다."
```

## 3. 시각화 선택 기준

### 3-1. 차트 선택 규칙

| 데이터 유형 | 차트 종류 | 이유 |
|---|---|---|
| 자산군별 비중 | 도넛 차트 (Donut Chart) | 전체 대비 비율 직관적 표현 |
| 수익률 추이 (시계열) | 라인 차트 (Line Chart) | 시간에 따른 변화 추적 |
| 종목별 수익률 비교 | 바 차트 (Bar Chart) | 항목 간 크기 비교 |
| 리밸런싱 목표 vs 현재 | 스택 바 차트 | 목표와 현재 비중 비교 |
| 리스크 등급 시각화 | 게이지 차트 (Gauge) | 단계적 위험도 표현 |

## 4. AI 인사이트 생성 규칙

### 4-1. 인사이트 생성 조건

```
트리거: 사용자가 "인사이트 생성" 버튼 클릭 시
조건: 자산이 1개 이상 등록되어 있어야 생성 가능
입력 데이터: 포트폴리오 전체 현황 (자산군별 비중, 수익률, 리스크 지표)
```

### 4-2. 인사이트 구성 항목

Google Gemini API에 전달하는 시스템 프롬프트 구조:

```
SDK: @google/generative-ai (gemini-2.5-flash 모델)
환경변수: GOOGLE_AI_API_KEY

당신은 개인 자산관리 전문 AI 어드바이저입니다.
아래 포트폴리오 데이터를 분석하여 다음 4가지 항목으로 인사이트를 제공하세요.
절대로 특정 종목의 매수/매도를 직접적으로 권유하지 마세요.
모든 분석은 현재 데이터 기반이며, 투자 결정은 사용자 본인의 판단임을 명시하세요.

[인사이트 구성]
1. 포트폴리오 요약: 전체 자산 현황을 2~3문장으로 요약
2. 리스크 경고: 편중/위험 감지 시 구체적 경고 메시지 (없으면 "안정적" 표시)
3. 리밸런싱 제안: 목표 비중 대비 조정이 필요한 항목과 구체적 비율 제시
4. 시장 상황 조언: 현재 시장 트렌드를 반영한 포트폴리오 관리 조언

[출력 형식]
- 한 줄 요약: 전체를 한 문장으로 압축 (summary 필드)
- 상세 내용: 위 4가지 항목을 각각 분리하여 제공 (details 필드)
- 어조: 친근하지만 전문적, 과도한 낙관/비관 금지
- 분량: 요약 1문장, 상세 각 항목 2~4문장
- 응답은 반드시 JSON으로만 출력 (마크다운 코드 블록 제거 후 파싱)
```

### 4-3. 인사이트 표시 규칙

```
기본 표시: 한 줄 요약만 노출
펼치기 클릭 시: 4가지 항목 상세 내용 노출
생성 시간: 인사이트 생성 일시 표시
갱신 조건: 자산 변경 후 기존 인사이트는 "업데이트 필요" 상태로 표시
면책 문구: "본 인사이트는 참고용이며, 투자 결정의 책임은 본인에게 있습니다."
```

### 4-4. 시장 상황 조언 생성 규칙

```
조언 방식: Gemini 모델의 학습 데이터 기반 거시경제 흐름 조언 (실시간 웹서치 미사용)
조언 범위: 거시경제 흐름 기반 포트폴리오 관리 조언에 한정
특정 종목 추천 금지: 개별 종목 매수/매도 직접 권유 절대 금지
```

## 5. 리포트 구성 규칙

### 5-1. 월간 리포트 구성

```
1. 이달의 총 자산 변동
   - 월초 대비 월말 총 자산 변화 (금액 + 수익률)

2. 자산군별 성과
   - 주식 / 코인 / 현금 각각의 이달 수익률

3. 베스트 / 워스트 종목
   - 이달 수익률 상위 1개, 하위 1개

4. 리스크 변화 추이
   - 이달 리스크 등급 변화 (전월 대비)

5. 다음 달 관리 포인트
   - 리밸런싱 필요 여부
   - 만기 도래 예금 안내
```

### 5-2. 리포트 시각화

```
월간 자산 변동: 라인 차트 (월별 총 자산 추이)
자산군별 성과: 바 차트 (자산군별 월 수익률 비교)
베스트/워스트: 강조 카드 UI
```

## 6. 데이터 확장성 규칙 (API 연동 대비)

### 6-1. 데이터 레이어 추상화 원칙

```
현재가 조회는 반드시 추상화된 인터페이스를 통해 호출
interface PriceProvider {
  getStockPrice(symbol: string, exchange: string): Promise<number>
  getCryptoPrice(symbol: string): Promise<number>
  getExchangeRate(from: string, to: string): Promise<number>
}

1차 구현: ManualPriceProvider (사용자 직접 입력)
확장 구현: ApiPriceProvider (증권사 API, 거래소 API 연동)
```

### 6-2. 확장 가능한 거래소 목록

```
주식:
  국내: 한국투자증권 API, 키움증권 API
  해외: Alpha Vantage, Yahoo Finance

코인:
  국내: 업비트 Open API, 빗썸 API
  해외: Binance API, CoinGecko API

환율:
  한국수출입은행 환율 API
  ExchangeRate-API
```
